REST API ส่วนใหญ่คือการ create, read, update และ delete กระทำต่อ resource ต่าง ๆ ศิลปะของมันคือการจับคู่การกระทำเหล่านั้นเข้ากับ HTTP methods เพื่อให้ทั้ง client, proxy และ logic การ retry ของคุณเองทำงานได้อย่างถูกต้อง

## CRUD สู่ HTTP

การจับคู่นี้ไม่ได้เกิดขึ้นแบบสุ่ม แต่ละ method มาพร้อมกับการรับประกันเรื่อง **safety** (มันเปลี่ยนแปลงข้อมูลหรือไม่?) และ **idempotency** (การทำซ้ำให้ผลเหมือนกับการทำครั้งเดียวหรือไม่?) การรับประกันเหล่านี้คือสิ่งที่ทำให้ client สามารถ retry คำขอที่หายไปได้อย่างปลอดภัย หรือทำให้ proxy แคช response ได้

## โมดูลนี้ครอบคลุมอะไรบ้าง

- **GET & POST** — การอ่านอย่างปลอดภัยและการสร้าง resource
- **PUT, PATCH & DELETE** — การแทนที่ การอัปเดตบางส่วน และการลบ
- **Idempotency & safety** — คุณสมบัติที่ทำให้การ retry ปลอดภัย
- **Bulk & async** — การทำงานแบบ batch และงานที่ใช้เวลานาน

<Callout title="การเลือก method คือคำสัญญา">
การเลือก method ไม่ใช่แค่การติดป้ายชื่อ แต่เป็นการสัญญากับ client ว่าการกระทำนั้นจะทำงานอย่างไร `GET` ที่เปลี่ยนแปลง state หรือ `DELETE` ที่ไม่ idempotent จะทำลายทั้ง cache, retry และความเชื่อมั่น จงเคารพสัญญาที่แต่ละ method สื่อถึง
</Callout>
