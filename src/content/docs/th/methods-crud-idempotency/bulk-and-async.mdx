API จริงบางครั้งจำเป็นต้องเปลี่ยนแปลง resource จำนวนมากในคราวเดียว หรือเริ่มงานที่ใช้เวลานานกว่าที่คำขอหนึ่งควรรอ ทั้งสองกรณีนี้ยืดโมเดลแบบหนึ่งคำขอต่อหนึ่ง resource ออกไป จึงต้องการการออกแบบอย่างตั้งใจ

## การทำงานแบบ Bulk

ในการกระทำต่อ item หลายตัวในการเรียกครั้งเดียว ให้ `POST` ไปยัง batch endpoint พร้อม list และคืน **ผลลัพธ์รายตัว** (per-item result) เพื่อให้ผู้เรียกรู้ได้อย่างแม่นยำว่าอะไรสำเร็จบ้าง:

```http
POST /articles/batch HTTP/1.1
Content-Type: application/json

{ "items": [ { "title": "A" }, { "title": "" } ] }
```

response รายงานผลลัพธ์ของ item แต่ละตัวแทนที่จะเป็น status รวมเพียงค่าเดียว — ความสำเร็จบางส่วน (partial success) คือเรื่องปกติสำหรับ batch:

export const batchCode = `function processBatch(items) {
  return items.map((item, i) => {
    if (!item.title) {
      return { index: i, status: 422, error: 'title is required' };
    }
    return { index: i, status: 201, id: 1000 + i };
  });
}

const results = processBatch([{ title: 'A' }, { title: '' }, { title: 'C' }]);
console.log(JSON.stringify(results, null, 2));
const ok = results.filter((r) => r.status < 400).length;
console.log(ok + '/' + results.length + ' succeeded');`;

## งานที่ใช้เวลานาน (async)

เมื่อ operation ไม่สามารถทำเสร็จภายในคำขอหนึ่งได้ (การ export ขนาดใหญ่ การ transcode วิดีโอ) อย่าค้างการเชื่อมต่อเอาไว้ ให้รับงานนั้นไว้แล้วคืน `202 Accepted` พร้อมลิงก์ไปยัง **status resource** ที่ client สามารถ poll ได้:

export const asyncDiagram = `sequenceDiagram
  participant C as Client
  participant S as API
  C->>S: POST /exports
  S-->>C: 202 Accepted + Location: /exports/job-1
  C->>S: GET /exports/job-1
  S-->>C: 200 OK { status: "running" }
  C->>S: GET /exports/job-1
  S-->>C: 200 OK { status: "done", result: "/files/x" }`;

<Mermaid code={asyncDiagram} title="202 Accepted พร้อม status resource ที่ poll ได้" />

ตัวงานเองกลายเป็น resource: `POST /exports` สร้างมันขึ้นมา, `GET /exports/{id}` รายงานความคืบหน้า และ response สุดท้ายลิงก์ไปยังผลลัพธ์ที่ผลิตออกมา

<Callout title="ทำให้งานกลายเป็น resource">
ทั้งสองแพตเทิร์นพึ่งพาแนวคิดเดียวกัน: เมื่อคำขอเดียวไม่สามารถสื่อ operation ได้ ให้จำลอง operation นั้นเป็น resource batch คืนผลลัพธ์รายตัว ส่วน async job กลายเป็นสิ่งที่คุณสร้างขึ้นแล้ว poll ตามทีหลัง คุณยังคงความเป็น RESTful ได้ด้วยการเปลี่ยนคำกริยา (verb) ให้เป็นคำนาม (noun)
</Callout>
